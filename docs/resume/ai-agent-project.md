# AI Agent 开发简历项目写法

## 项目标题

校园公开信息研究 Agent 与 EvalOps Runtime

## 简历 bullets

- 基于 LangGraph 实现 Planning、Tool Calling、Verification 和单次有界 Replan 闭环，接入公开检索、详情读取、截止时间线和证据比较工具，结论可追溯到原始来源。
- 使用 Django、Celery、MySQL、Redis 与 SSE 构建异步运行时，支持幂等创建、取消、断线续传、版本冻结 Replay、独立 Agent 队列及增量 RAG 索引。
- 以 Pydantic 定义工具契约和权限元数据，实现提示词注入隔离、SSRF 重定向防护，以及管理员审批后的来源诊断、重试与索引修复。
- 构建 120 条分层规划安全集与 40 条冻结检索集；零成本回归中规划结构有效率 100%、工具选择准确率 100%、越权选择 0，检索门槛 Recall@5≥95%、MRR≥90%。

## 面试讲解思路

1. 先讲业务问题：学校通知分散在官网、学院站、就业网和公众号链接里，人工筛选成本高、错过信息风险高。
2. 再讲 Agent 工作流：调度抓取 -> 列表发现 -> 正文抽取/OCR -> 去重 -> AI 摘要分类 -> 重要度排序 -> RAG chunk 索引。
3. 展示 `/agent/`：说明数据规模、运行记录、评测结果、成本控制和自愈机制。
4. 展示 `/ask/`：用示例问题验证检索、引用、流式回答、token/费用统计和 fallback。
5. 最后讲工程约束：2C2G 服务器、Docker Compose、MySQL/Redis/Meilisearch、预算熔断、后台权限和公开页面脱敏。

## 为什么这样写

- 大厂 AI Agent 实习更看重工程闭环：数据接入、RAG、工具调用、评测、可观测、上线和成本控制，比单纯调用大模型 API 更有区分度。
- 项目有真实用户入口和真实数据规模，能证明你做过线上系统，而不是一次性 demo。
- 简历 bullet 用数字和机制表达结果，面试时可以直接打开 `/agent/` 和 `/ask/` 逐项验证。

## 可以主动承认的限制

- 当前公开入口已启用 HTTPS；会话长期记忆仍默认关闭，直到以登录用户、显式授权和可删除的数据生命周期实现。
- 尚未完成付费模型端到端人工答案质量对照，不声称答案准确率提升。
- 选择单 Agent 状态图而非多 Agent，以降低 2C2G 部署的延迟、成本和重复调用。
- 公众号和社媒账号主页没有做自动监控，第一版只处理具体公开链接，避免抓取合规和反爬风险扩大。
